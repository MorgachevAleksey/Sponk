# Spring Boot — универсальная база знаний

## 1. Что такое веб-приложение

Веб-приложение — это программа, с которой взаимодействуют через HTTP-запросы.

Пользователь или другой сервис отправляет запрос на сервер, а сервер возвращает ответ.

### Основные части
- клиент — тот, кто отправляет запрос;
- сервер — тот, кто принимает и обрабатывает запрос;
- HTTP — протокол обмена сообщениями между ними.

---

## 2. Что такое API

API — это интерфейс взаимодействия между частями программы или между разными программами.

Проще говоря, API — это набор правил:
- как к системе обратиться;
- какие данные передать;
- какой ответ получить;
- в каком формате всё это происходит.

---

## 3. Что такое REST

REST — это стиль построения API для работы через HTTP.

REST API обычно использует HTTP-методы:
- `GET` — получить данные;
- `POST` — создать данные;
- `PUT` — обновить данные;
- `DELETE` — удалить данные.

### Идея REST
В REST мы работаем не с “действиями”, а с ресурсами.

Ресурс — это сущность, с которой работает API:
- пользователи;
- задачи;
- статьи;
- заметки;
- заказы.

---



### Главная идея Spring
Spring берёт на себя большую часть технической рутины, чтобы разработчик мог сосредоточиться на логике приложения.

---

## 5. Что такое Spring Boot

Spring Boot — это надстройка над Spring, которая упрощает запуск и настройку приложения.

Он нужен, чтобы:
- не писать много ручной конфигурации;
- быстро поднимать проект;
- подключать готовые зависимости через стартеры;
- автоматически настраивать типовые вещи.

### Что делает Spring Boot
Spring Boot помогает собрать приложение в рабочее состояние почти сразу после подключения нужных зависимостей.

---

## 6. Что такое автоконфигурация

Автоконфигурация — это механизм, при котором Spring Boot сам настраивает приложение в зависимости от того, что подключено в проект.

### Простыми словами
Если в проекте есть web-зависимость, Spring Boot понимает, что нужно поднять web-часть.

Если есть работа с базой данных, Spring Boot настраивает инфраструктуру для доступа к данным.

### Зачем это нужно
Автоконфигурация экономит время и снижает количество ручной настройки.

---

## 7. Что такое стартеры

Стартер — это готовый набор зависимостей для конкретной задачи.

Например:
- web-стартер — для REST и HTTP;
- validation-стартер — для проверки входных данных;
- data-jpa-стартер — для работы с БД;
- test-стартер — для тестирования.

### Почему это удобно
Вместо десятков отдельных библиотек подключается один стартер, который приносит нужный набор возможностей.

---

## 8. Что такое MVC

MVC — это архитектурный паттерн, который делит приложение на три части:

- Model — данные и логика данных;
- View — то, что видит пользователь;
- Controller — принимает запрос и управляет потоком обработки.

### В веб-приложениях на Spring
Чаще всего MVC используется так:
- Controller — принимает HTTP-запрос;
- Service — содержит бизнес-логику;
- Repository — работает с данными;
- Entity — описывает данные в базе;
- DTO — описывает данные для обмена через API.

---

## 9. Почему важно разделять слои

Разделение слоёв делает проект понятным и поддерживаемым.

### Controller
Принимает запрос и отдаёт ответ.

### Service
Содержит бизнес-логику.

### Repository
Работает с данными и базой.

### Entity
Описывает структуру хранения данных.

### DTO
Описывает данные, которые приходят и уходят через API.

### Зачем это нужно
Так код:
- проще читать;
- легче тестировать;
- удобнее расширять;
- проще сопровождать.

---

## 10. Что такое dependency injection

Dependency Injection — это подход, при котором объект получает нужные ему зависимости извне, а не создаёт их сам.

### Простыми словами
Класс не должен сам искать себе всё, что ему нужно.
Он получает это через Spring.

### Зачем это нужно
Это делает код:
- гибким;
- тестируемым;
- менее связанным;
- удобным для замены компонентов.

---

## 11. Что такое IoC

IoC — Inversion of Control — это принцип, при котором контроль над созданием и управлением объектами передаётся фреймворку.

### Простыми словами
Не код управляет контейнером, а контейнер управляет кодом.

Spring сам создаёт нужные объекты, связывает их и живёт по своей схеме.

---

## 12. Что такое bean

Bean — это объект, которым управляет Spring.

### Простыми словами
Если Spring создал объект, сохранил его в контексте и управляет его жизненным циклом, это bean.

---

## 13. Что такое контекст Spring

Контекст Spring — это окружение, в котором живут beans и через которое Spring управляет приложением.

### Зачем это нужно
Через контекст Spring знает:
- какие объекты существуют;
- как они связаны;
- когда их создавать;
- как их внедрять.

---

## 14. Что такое `@SpringBootApplication`

Это главная аннотация стартового класса Spring Boot-приложения.

Она объединяет:
- конфигурацию;
- автоконфигурацию;
- сканирование компонентов.

### Простыми словами
Она говорит Spring:
“это главное приложение, найди мои компоненты и собери всё вместе”.

---

## 15. Что делает стартовый класс

Стартовый класс — это точка входа приложения.

Обычно в нём есть `main`, внутри которого вызывается `SpringApplication.run(...)`.

### Смысл
Именно с этого запуска Spring начинает собирать приложение и поднимать все нужные компоненты.

---

## 16. Что такое `@RestController`

`@RestController` — это контроллер для REST API.

Он нужен, когда мы хотим возвращать не страницу, а данные.

### Простыми словами
Это класс, который принимает HTTP-запросы и возвращает ответ прямо в тело HTTP-ответа.

---

## 17. Что такое `@Controller`

`@Controller` — это контроллер, который чаще используется для страниц и шаблонов.

Если приложение отдаёт HTML-страницы, обычно используют его.

Если приложение отдаёт JSON или другой API-ответ, обычно используют `@RestController`.

---

## 18. Что такое `@RequestMapping`

`@RequestMapping` задаёт маршрут для контроллера или метода.

### Простыми словами
Это способ сказать:
“вот этот URL должен обрабатываться вот этим кодом”.

---

## 19. Что такое `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`

Это короткие и удобные варианты `@RequestMapping` для конкретных HTTP-методов.

- `@GetMapping` — получить данные;
- `@PostMapping` — создать;
- `@PutMapping` — обновить;
- `@DeleteMapping` — удалить.

---

## 20. Что такое endpoint

Endpoint — это конкретный адрес в API.

Пример:
- `GET /api/items`
- `POST /api/items`
- `GET /api/items/1`

### Простыми словами
Endpoint — это точка входа, куда приходит запрос.

---

## 21. Что такое HTTP-статусы

HTTP-статус показывает результат обработки запроса.

### Основные статусы
- `200 OK` — всё хорошо;
- `201 Created` — объект создан;
- `400 Bad Request` — ошибка в запросе;
- `404 Not Found` — ресурс не найден;
- `500 Internal Server Error` — внутренняя ошибка сервера.

### Зачем это нужно
Статус-коды помогают клиенту понять, что произошло.

---

## 22. Что такое JSON

JSON — это текстовый формат для обмена данными.

### Почему он важен
JSON читается человеком, легко передаётся по сети и часто используется в REST API.

---

## 23. Что такое DTO

DTO — Data Transfer Object.

Это объект для передачи данных между слоями или через API.

### Зачем он нужен
DTO помогает:
- не светить внутреннюю структуру сущностей;
- отделить модель хранения от модели передачи;
- делать API чище и безопаснее.

---

## 24. Что такое Entity

Entity — это класс, который описывает объект, сохраняемый в базе данных.

### Простыми словами
Entity — это форма данных для хранения.

---

## 25. Что такое Repository

Repository — это слой доступа к данным.

Он отвечает за:
- сохранение;
- чтение;
- обновление;
- удаление;
- поиск.

### Простыми словами
Repository — это мост между приложением и базой данных.

---

## 26. Что такое Service

Service — это слой бизнес-логики.

Именно здесь находится смысл работы приложения:
- что делать;
- в каком порядке;
- какие проверки применять;
- что возвращать.

### Почему не в контроллере
Контроллер должен быть тонким, чтобы не превращаться в свалку логики.

---

## 27. Что такое validation

Validation — это проверка входных данных.

### Зачем нужна
Чтобы не принимать:
- пустые значения;
- слишком длинные строки;
- некорректные форматы;
- недопустимые комбинации данных.

---

## 28. Что такое exception handling

Exception handling — это обработка ошибок.

### Зачем нужна
Если что-то пошло не так, приложение должно:
- не падать;
- вернуть понятный ответ;
- объяснить проблему клиенту.

---

## 29. Что такое `@ControllerAdvice`

Это способ централизованно обрабатывать ошибки во всём приложении.

### Простыми словами
Вместо того чтобы ловить ошибки в каждом контроллере отдельно, можно описать общую точку обработки.

---

## 30. Что такое Spring Data JPA

Spring Data JPA — это удобный способ работать с базой данных через Java-объекты.

### Идея
Ты описываешь сущность и репозиторий, а Spring помогает с CRUD-операциями и запросами.

---

## 31. Что такое ORM

ORM — Object-Relational Mapping.

Это способ связать объекты Java и таблицы базы данных.

### Простыми словами
Java-объект превращается в запись таблицы, и наоборот.

---






## 38. Как обычно строится разработка Spring Boot-проекта

Типичный порядок такой:
1. определить цель проекта;
2. создать каркас;
3. подключить зависимости;
4. сделать стартовый класс;
5. создать контроллер;
6. добавить сервис;
7. добавить repository;
8. добавить entity;
9. добавить DTO;
10. добавить validation;
11. добавить обработку ошибок;
12. подключить БД;
13. проверить через Postman;
14. написать тесты;
15. оформить README;
16. подготовить проект к сдаче или демонстрации.
